# KB-010 — PostgreSQL + pgAdmin

## Status
- **Coluna atual:** Done
- **Prioridade:** Média
- **Criado em:** 2026-04-20

## Descrição

Deployar uma instância compartilhada de PostgreSQL para ser utilizada por todas as aplicações do cluster que necessitem de banco de dados relacional. Junto ao PostgreSQL, deployar o pgAdmin 4 como ferramenta de administração.

**Estratégia adotada:** uma única instância PostgreSQL com múltiplos databases — um por aplicação. Leve e adequado para o hardware disponível (7.6 GB RAM).

## Critérios de Aceite

- [ ] PostgreSQL 17 deployado via ArgoCD (App of Apps) usando Bitnami Helm chart
- [ ] Dados persistidos em `/opt/homelab/data/postgresql` no host (hostPath)
- [ ] `make uninstall` preserva o diretório `/opt/homelab/data/` (não apaga os dados)
- [ ] Senha do superusuário `postgres` gerenciada via SealedSecret (não exposta no Git)
- [ ] pgAdmin 4 deployado via ArgoCD
- [ ] pgAdmin acessível via `pgadmin.bisnaguete.xyz` com HTTPS
- [ ] Login do pgAdmin: `admin@bisnaguete.xyz` com senha via SealedSecret
- [ ] pgAdmin conectado ao PostgreSQL via conexão pré-configurada (`servers.json`)
- [ ] PostgreSQL não exposto externamente (ClusterIP apenas)
- [ ] Documentado como criar um novo database para uma nova aplicação

## Estratégia de Databases

- Uma instância PostgreSQL, múltiplos databases
- Cada aplicação que precisar de banco recebe seu próprio database isolado
- Convenção de nomenclatura: `<nome-do-app>_db` (ex: `nextcloud_db`, `paperless_db`)
- Usuário dedicado por database (menor privilégio)
- Databases criados pelo card de cada aplicação — o KB-010 entrega apenas a instância vazia

## Notas Técnicas

- Bitnami PostgreSQL Helm chart (`bitnami/postgresql`)
- Versão fixada: **PostgreSQL 17**
- Persistência via `hostPath` em `/opt/homelab/data/postgresql` — fora do caminho do K3s, sobrevive ao `make uninstall`
- O diretório raiz `/opt/homelab/` será criado pelo Ansible (role `common`) antes do K3s subir
- Estrutura de diretórios no host:
  ```
  /opt/homelab/
  ├── data/          ← dados de apps
  │   └── postgresql/
  └── media/         ← futuramente: filmes, livros, torrents, etc.
  ```
- pgAdmin configurado com conexão pré-definida via `servers.json` montado como ConfigMap
- Sync wave: 2 (sobe junto com o ArgoCD, antes das apps)

## Estrutura no Repositório

```
argocd/apps/
├── postgresql.yaml

apps/
├── postgresql/

    ├── ingress.yaml
    ├── servers-configmap.yaml    # conexão pré-configurada com o postgres

```
