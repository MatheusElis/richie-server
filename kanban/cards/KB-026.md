# KB-026 — Integrar apps existentes com Authentik (Forward Auth)

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Média
- **Criado em:** 2026-04-22

## Descrição

Adicionar o middleware ForwardAuth do Authentik nos ingresses das aplicações já em produção: Glance, pgAdmin e Joplin. Elimina acesso não autenticado e centraliza o controle de acesso.

## Detalhes Técnicos

### Apps afetadas

| App | Namespace | Ingress | Método |
|---|---|---|---|
| Glance | glance | `home.bisnaguete.xyz` | Forward Auth (sem auth própria — Authentik protege) |
| pgAdmin | pgadmin | `pgadmin.bisnaguete.xyz` | Forward Auth + header `REMOTE_USER` para auto-login |

### Alterações por app

**Glance:**
- Adicionar annotation no ingress: `traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd`
- Sem auth própria → Authentik é a única camada

**pgAdmin:**
- Adicionar annotation de Forward Auth no ingress
- Configurar pgAdmin para aceitar `REMOTE_USER` header (proxy auth)
- Evita dupla autenticação: Authentik autentica, pgAdmin aceita automaticamente

### Manifests a alterar

- `apps/glance/ingress.yaml`
- `apps/pgadmin/ingress.yaml`
- `apps/pgadmin/deployment.yaml` (env var para proxy auth)

### Configuração no Authentik

Para cada app, criar no Authentik:
1. **Application** com slug correspondente
2. **Provider** tipo Proxy (Forward Auth — single application)
3. Vincular ao **Outpost** existente

## Critérios de Aceite

- [ ] Glance: acesso via `home.bisnaguete.xyz` exige login no Authentik
- [ ] pgAdmin: login automático após autenticação no Authentik (sem tela de login do pgAdmin)
- [ ] Todas as apps: ArgoCD Synced + Healthy após alterações

## Notas Técnicas

- A alteração é mínima por app — basicamente adicionar annotation no ingress
- O pgAdmin suporta `REMOTE_USER` nativamente via config `AUTHENTICATION_SOURCES: ['webserver']`

## Dependências

- KB-024 (Authentik)
