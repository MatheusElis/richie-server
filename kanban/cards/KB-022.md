# KB-022 — Joplin Server (sincronização de notas)

## Status
- **Coluna atual:** Review
- **Prioridade:** Média
- **Criado em:** 2026-04-21

## Descrição

Deploy do Joplin Server para sincronização de notas entre dispositivos. Usa o PostgreSQL já existente no cluster como banco de dados.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `joplin/server:3.5.2` |
| Porta | 22300 |
| Volumes | Nenhum volume de dados (usa PostgreSQL) |
| Env vars (obrigatórias) | `APP_BASE_URL`, `APP_PORT`, `DB_CLIENT=pg`, `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DATABASE`, `POSTGRES_HOST`, `POSTGRES_PORT` |
| Credenciais admin | `admin@localhost` / `admin` (padrão fixo — alterar via web UI após 1º login) |
| Ingress | `https://joplin.bisnaguete.xyz` |

### Manifests

- `apps/joplin/deployment.yaml`
- `apps/joplin/service.yaml`
- `apps/joplin/ingress.yaml`
- `apps/joplin/sealed-secret.yaml` (credenciais do PostgreSQL)
- `argocd/apps/joplin.yaml`

### Nota sobre credenciais admin

O Joplin Server **não suporta** configuração de admin via env var. O usuário padrão é `admin@localhost` / `admin`. Após o primeiro deploy, alterar email e senha via web UI.

### Preparação no PostgreSQL

A criação do database e usuário será automatizada via **init container** no Deployment do Joplin. O init container usa a imagem `postgres:17` e executa:
```sql
CREATE DATABASE joplin;
CREATE USER joplin WITH PASSWORD 'senha-aqui';
GRANT ALL PRIVILEGES ON DATABASE joplin TO joplin;
```
As credenciais do PostgreSQL (host, user, password) vêm do SealedSecret do Joplin.

## Critérios de Aceite

- [ ] Init container cria database `joplin` no PostgreSQL automaticamente
- [ ] Pod running no namespace `joplin`
- [ ] Web UI acessível via `https://joplin.bisnaguete.xyz`
- [ ] Login com credenciais padrão funcional
- [ ] Sincronização de notas via app Joplin desktop/mobile
- [ ] ArgoCD: Synced + Healthy
