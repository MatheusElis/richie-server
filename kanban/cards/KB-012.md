# KB-012 — Quebrar dependência cíclica ArgoCD ↔ Sealed Secrets

## Status
- **Coluna atual:** To Do
- **Prioridade:** Alta
- **Criado em:** 2026-04-21

## Descrição

### Problema

Dependência circular no `make install` do zero:

```
ArgoCD precisa do Sealed Secrets (para processar SealedSecret da senha admin)
    ↕
Sealed Secrets controller é instalado PELO ArgoCD (App of Apps, wave 0)
```

A task "Aplicar SealedSecret da senha do admin ArgoCD" falha com:
```
no matches for kind "SealedSecret" in version "bitnami.com/v1alpha1"
ensure CRDs are installed first
```

### Solução

**Sealed Secrets é infraestrutura base** — deve ser instalado pelo Ansible (via Helm), não pelo ArgoCD.

### Mudanças necessárias

#### 1. `ansible/inventory/group_vars/all.yml`
- Adicionar variável `sealed_secrets_chart_version: "2.16.2"` para fixar a versão do chart Helm

#### 2. `ansible/roles/sealed-secrets/tasks/main.yml`
- Adicionar `helm repo add` do repositório bitnami-labs
- Adicionar `helm upgrade --install` do controller no namespace `kube-system`
- Aguardar controller estar pronto (`rollout status`)
- Manter a lógica existente de restaurar/exportar a chave privada

#### 3. `argocd/apps/sealed-secrets.yaml`
- **Remover** — Sealed Secrets não será mais gerenciado pelo ArgoCD

#### 4. `ansible/roles/argocd/tasks/main.yml`
- Reordenar tasks:
  - Helm install ArgoCD
  - Aplicar SealedSecret da senha admin (CRD já existe via role anterior)
  - Anotar + deletar + recriar `argocd-secret` para forçar reconciliação
  - Reiniciar argocd-server
  - Aplicar root-app (App of Apps)
  - Validar pods

### Fluxo corrigido

```
site.yml:
  1. common          → configura OS
  2. k3s             → instala K3s
  3. sealed-secrets  → helm install controller + restaura chave ← MUDANÇA
  4. argocd:
     a. helm install ArgoCD
     b. ✅ kubectl apply SealedSecret (CRD já existe!)
     c. anotar + reconciliar argocd-secret
     d. reiniciar argocd-server
     e. aplicar root-app (App of Apps)
     f. validar pods
```

## Critérios de Aceite

- [ ] Sealed Secrets controller instalado via Ansible (Helm) no role `sealed-secrets`
- [ ] Variável `sealed_secrets_chart_version` em `group_vars/all.yml`
- [ ] `argocd/apps/sealed-secrets.yaml` removido
- [ ] Role `argocd` com tasks reordenadas (SealedSecret aplicado após Helm install)
- [ ] `make install` do zero funciona sem erros
- [ ] Controller do Sealed Secrets rodando em `kube-system`
- [ ] SealedSecret da senha admin processado corretamente
- [ ] ArgoCD acessível com senha configurada

## Notas Técnicas

- O Sealed Secrets chart Helm atual é versão `2.16.2` (extraído de `argocd/apps/sealed-secrets.yaml`)
- O release name deve ser `sealed-secrets-controller` com `fullnameOverride` para manter compatibilidade
- Namespace: `kube-system` (mesmo de antes)
- A chave privada continua em `~/.homelab/sealed-secrets-key.yaml`
- As sync-waves das outras apps no ArgoCD não precisam mudar
