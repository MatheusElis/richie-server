# KB-018 — Sonarr (gerenciador de séries)

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Média
- **Criado em:** 2026-04-21

## Descrição

Deploy do Sonarr para gerenciamento automático de séries de TV.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `linuxserver/sonarr:4.0.17` |
| Porta | 8989 |
| Volumes | `/config` (persistente), `/tv` → `/opt/homelab/data/media/tv`, `/downloads` → `/opt/homelab/data/media/downloads` |
| Env vars | `PUID`, `PGID`, `TZ`, `SONARR__AUTH__METHOD=External`, `SONARR__AUTH__REQUIRED=DisabledForLocalAddresses` |
| Ingress | `https://sonarr.bisnaguete.xyz` |

### Manifests

- `apps/sonarr/deployment.yaml`
- `apps/sonarr/service.yaml`
- `apps/sonarr/ingress.yaml`
- `apps/sonarr/pv.yaml` (config)
- `argocd/apps/sonarr.yaml`

## Critérios de Aceite

- [ ] Pod running no namespace `sonarr`
- [ ] Web UI acessível via `https://sonarr.bisnaguete.xyz`
- [ ] Consegue ver pasta de séries e downloads
- [ ] ArgoCD: Synced + Healthy

## Notas Técnicas

- **Autenticação:** `AuthenticationMethod=External` — confia no header `Remote-User` do Authentik. Sem tela de login própria. Annotation no ingress: `traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd`
- **Integração com Prowlarr:** após deploy, configurar Prowlarr como gerenciador de indexadores via Settings → Indexers na web UI do Sonarr. Mesma configuração definida no Radarr.

## Dependências

- KB-014 (volume compartilhado de mídia)
- KB-024 (Authentik — Forward Auth + header Remote-User)
