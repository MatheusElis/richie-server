- **Coluna atual:** Backlog

## Descrição

Deploy do Prowlarr como gerenciador centralizado de indexadores para o stack *arr.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `linuxserver/prowlarr:1.34.1` |
| Porta | 9696 |
| Volumes | `/config` (persistente) |
| Env vars | `PUID`, `PGID`, `TZ`, `PROWLARR__AUTH__METHOD=External`, `PROWLARR__AUTH__REQUIRED=DisabledForLocalAddresses` |
| Ingress | `https://prowlarr.bisnaguete.xyz` |
| Dependências | Nenhuma (integra com Radarr/Sonarr via API) |

### Manifests

- `apps/prowlarr/deployment.yaml`
- `apps/prowlarr/service.yaml`
- `apps/prowlarr/ingress.yaml`
- `apps/prowlarr/pv.yaml` (config)
- `argocd/apps/prowlarr.yaml`

### Autenticação

Usa `AuthenticationMethod=External` — confia no header `Remote-User` enviado pelo Authentik via Forward Auth. Sem tela de login própria.

Annotation no ingress:
```yaml
traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd
```
