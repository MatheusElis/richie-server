# KB-007 — cert-manager + Cloudflare DNS-01

## Status
- **Coluna atual:** Done
- **Prioridade:** Alta
- **Criado em:** 2026-04-20

## Descrição

Instalar e configurar o cert-manager para emissão automática de certificados TLS válidos (Let's Encrypt) usando o DNS-01 challenge via API do Cloudflare.

Isso permite HTTPS real com certificado wildcard `*.bisnaguete.xyz` em todos os serviços do homelab **sem expor nenhuma porta para a internet**.

## Critérios de Aceite

- [ ] cert-manager instalado via ArgoCD (App of Apps — wave 1)
- [ ] `ClusterIssuer` `letsencrypt-staging` criado e funcional
- [ ] `ClusterIssuer` `letsencrypt-prod` criado e funcional
- [ ] Cloudflare API Token armazenado como SealedSecret no cluster
- [ ] Certificado wildcard `*.bisnaguete.xyz` emitido com sucesso (staging primeiro, depois prod)
- [ ] Renovação automática validada (cert-manager renova aos 30 dias do vencimento)
- [ ] Traefik consumindo o certificado automaticamente para todos os Ingresses

## Como Funciona

```
cert-manager detecta Ingress com anotação de TLS
    └── Cria registro TXT _acme-challenge.bisnaguete.xyz no Cloudflare via API Token
        └── Let's Encrypt valida o TXT record
            └── Certificado *.bisnaguete.xyz emitido (válido por 90 dias)
                └── Armazenado como Secret no cluster
                    └── Traefik usa o certificado automaticamente ✅
```

## ClusterIssuers

### Estratégia de uso
1. Durante setup: usar `letsencrypt-staging` para validar o fluxo sem risco de rate limit
2. Após validação: trocar para `letsencrypt-prod` para certificado válido e confiável

```yaml
# Exemplo de uso no Ingress
annotations:
  cert-manager.io/cluster-issuer: "letsencrypt-prod"
```

## Token do Cloudflare

O API Token (já criado) será armazenado como SealedSecret:

```
Token do Cloudflare
    └── kubeseal encripta com ~/.homelab/sealed-secrets-cert.pem
        └── apps/cert-manager/sealed-secret.yaml  ← entra no Git com segurança
            └── cert-manager usa o token para o DNS-01 challenge
```

Permissões mínimas do token:
- `Zone → DNS → Edit`
- `Zone → Zone → Read`
- `Zone Resources → bisnaguete.xyz`

## Notas Técnicas

- O registro `*` (wildcard) no Cloudflare com DNS only já cobre todos os subdomínios
- O cert-manager cria e remove o registro TXT de validação automaticamente
- O certificado wildcard cobre todos os subdomínios — um único cert para todo o cluster
- Sem wildcard DNS record na zona não é possível usar certificado wildcard com DNS-01
