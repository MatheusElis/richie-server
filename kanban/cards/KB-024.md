### Preparação no PostgreSQL

Init container com imagem `postgres:17`:
```sql
CREATE DATABASE authentik;
CREATE USER authentik WITH PASSWORD 'senha-aqui';
GRANT ALL PRIVILEGES ON DATABASE authentik TO authentik;
```

### Integração com Traefik (Forward Auth)

Após o deploy, criar no Authentik:
1. **Provider** tipo Proxy (Forward Auth - single application ou domain-level)
2. **Outpost** integrado ao provider
3. **Middleware Traefik** (`traefik-middleware.yaml`) que aponta para o outpost:

```yaml
apiVersion: traefik.io/v1alpha1
kind: Middleware
metadata:
  name: authentik
  namespace: authentik
spec:
  forwardAuth:
    address: http://authentik-server.authentik.svc:9000/outpost.goauthentik.io/auth/traefik
    trustForwardHeader: true
    authResponseHeaders:
      - X-authentik-username
      - X-authentik-groups
      - X-authentik-email
      - X-authentik-name
      - X-authentik-uid
      - X-authentik-jwt
      - X-authentik-meta-jwks
      - X-authentik-meta-outpost
      - X-authentik-meta-provider
      - X-authentik-meta-app
      - X-authentik-meta-version
      - Remote-User
      - Remote-Groups
      - Remote-Email
      - Remote-Name
```

As apps que usam Forward Auth referenciam este middleware no ingress via annotation:
```yaml
traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd
```

## Notas Técnicas

- Authentik é a peça central de autenticação — deve ser deployado ANTES das apps que dependem dele
- O consumo de RAM (~550 MB) é significativo para o hardware. Monitorar após deploy
- A secret key é crítica — se perdida, sessões e tokens são invalidados. Guardar backup
- Registro DNS `auth.bisnaguete.xyz` precisa ser criado no Cloudflare
- Senha do admin é definida no primeiro acesso via `https://auth.bisnaguete.xyz/if/flow/initial-setup/`
- **Redis com persistência:** dados salvos em disco via `hostPath`. Se no futuro outra app precisar de Redis, pode reutilizar este deploy (acessar via `redis.authentik.svc:6379`)
