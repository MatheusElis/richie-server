# KB-015 — Transmission (cliente torrent)

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Média
- **Criado em:** 2026-04-21

## Descrição

Deploy do Transmission como cliente de download torrent no cluster.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `linuxserver/transmission:4.1.1` |
| Porta Web UI | 9091 |
| Porta Peers | 51413 (TCP + UDP) |
| Volumes | `/config` (persistente), `/downloads` → `/opt/homelab/data/media/downloads` |
| Env vars | `PUID`, `PGID`, `TZ` |
| Ingress | `https://transmission.bisnaguete.xyz` |

### Manifests

- `apps/transmission/deployment.yaml`
- `apps/transmission/service.yaml`
- `apps/transmission/ingress.yaml`
- `apps/transmission/pv.yaml` (config)
- `argocd/apps/transmission.yaml`

### Autenticação

Sem auth própria — protegido via Forward Auth do Authentik (middleware no ingress). Annotation:
```yaml
traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd
```

### Nota sobre portas

A porta 51413 será exposta via `hostPort` no Deployment para que peers externos consigam conectar. A liberação da porta no firewall será automatizada via Ansible (role `common`).

## Critérios de Aceite

- [ ] Pod running no namespace `transmission`
- [ ] Web UI acessível via `https://transmission.bisnaguete.xyz`
- [ ] Download de torrent funcional
- [ ] Porta 51413 acessível externamente (peers conectam)
- [ ] ArgoCD: Synced + Healthy

- KB-024 (Authentik — Forward Auth)
