## Descrição

Deploy do Transmission como cliente de download torrent no cluster.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `linuxserver/transmission:4.1.1` |
| Porta Web UI | 9091 |
| Porta Peers | 51413 (TCP + UDP) |
| Volumes | `/config` (persistente), `/downloads` → `/opt/homelab/data/media/downloads` |
| Env vars | `PUID`, `PGID`, `TZ`, `USER`, `PASS` (via SealedSecret) |
| Ingress | `https://transmission.bisnaguete.xyz` |

### Manifests

- `apps/transmission/deployment.yaml`
- `apps/transmission/service.yaml`
- `apps/transmission/ingress.yaml`
- `apps/transmission/pv.yaml` (config)
- `apps/transmission/sealed-secret.yaml` (USER/PASS)
- `argocd/apps/transmission.yaml`

### Nota sobre portas

A porta 51413 precisa ser exposta via `hostPort` ou `NodePort` para que peers externos consigam conectar.

## Critérios de Aceite

- [ ] Pod running no namespace `transmission`
- [ ] Web UI acessível via `https://transmission.bisnaguete.xyz`
- [ ] Download de torrent funcional
- [ ] ArgoCD: Synced + Healthy
