# KB-014 — Volume compartilhado de mídia

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-21

## Descrição

Criar estrutura de diretórios e PV/PVC compartilhado para apps de mídia. Todas as apps que manipulam arquivos (Transmission, Radarr, Sonarr, LazyLibrarian, Calibre, Filebrowser) devem acessar o mesmo volume base.

### Estrutura proposta

```
/opt/homelab/data/media/
├── downloads/        # Transmission deposita aqui
│   ├── complete/
│   └── incomplete/
├── movies/           # Radarr organiza aqui
├── tv/               # Sonarr organiza aqui
└── books/            # LazyLibrarian + Calibre-Web
```

### Manifests necessários

- `apps/media/` — Não precisa de PV/PVC (usando hostPath direto nos Deployments de cada app)
- A criação dos diretórios será feita via Ansible (role `common`)

### Decisão de design

Usar `hostPath` direto em cada Deployment. É a abordagem mais simples para single-node — cada pod monta diretamente o subdiretório que precisa.

## Critérios de Aceite

- [ ] Diretórios criados via Ansible (role common ou novo role)
- [ ] Cada app de mídia monta os subdiretórios necessários
- [ ] Permissões corretas (PUID/PGID consistente entre apps)

## Notas Técnicas

- As imagens LinuxServer.io usam `PUID`/`PGID` para rodar como usuário não-root
- Definir PUID=1000 / PGID=1000 (usuário `elis`) para todas as apps de mídia
- O Filebrowser terá acesso ao `/opt/homelab/data` inteiro para navegação geral

## Dependências

- Nenhuma
