# KB-014 — Volume compartilhado de mídia

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-21

## Descrição

Criar estrutura de diretórios e PV/PVC compartilhado para apps de mídia. Todas as apps que manipulam arquivos (Transmission, Radarr, Sonarr, LazyLibrarian, Calibre-Web, Filebrowser) devem acessar o mesmo volume base.

### Estrutura proposta

```
/opt/homelab/data/media/
├── downloads/        # Transmission deposita aqui
│   ├── complete/
│   └── incomplete/
├── movies/           # Radarr organiza aqui
├── tv/               # Sonarr organiza aqui
└── books/            # LazyLibrarian + Calibre-Web
```

### Manifests necessários

- `apps/media/pv.yaml` — PersistentVolume apontando para `/opt/homelab/data/media`
- `apps/media/pvc.yaml` — PersistentVolumeClaim no namespace `media` (ou usar PVCs por namespace com mesmo hostPath)

### Decisão de design

Como múltiplos pods em namespaces diferentes precisam acessar os mesmos arquivos, usar `hostPath` direto em cada Deployment é mais simples do que PV/PVC (que são 1:1 por namespace). Alternativa: criar PV/PVC por namespace apontando para o mesmo hostPath.

## Critérios de Aceite

- [ ] Diretórios criados via Ansible (role common ou novo role)
- [ ] Cada app de mídia monta os subdiretórios necessários
- [ ] Permissões corretas (PUID/PGID consistente entre apps)

## Notas Técnicas

- As imagens LinuxServer.io usam `PUID`/`PGID` para rodar como usuário não-root
- Definir um PUID/PGID consistente (ex: 1000:1000) para todas as apps de mídia
- O Filebrowser terá acesso ao `/opt/homelab/data` inteiro para navegação geral
