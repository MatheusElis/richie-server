# KB-006 — Ingress com Traefik

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-20

## Descrição

Configurar o Traefik v3 como Ingress Controller do cluster, gerenciado via ArgoCD. Responsável por rotear o tráfego local por subdomínio e redirecionar HTTP → HTTPS globalmente.

O Traefik nativo do K3s é desabilitado no Ansible (KB-002) e substituído por esta instalação controlada via GitOps.

## Domínio

- **Domínio:** `bisnaguete.xyz`
- **Padrão de subdomínios:** `<app>.bisnaguete.xyz → 192.168.31.55`
- **DNS:** registros A no Cloudflare com proxy **desligado** (`DNS only`) apontando para `192.168.31.55`

## Critérios de Aceite

- [ ] Traefik v3 instalado via ArgoCD (App of Apps — wave 1)
- [ ] `IngressClass` configurado como padrão do cluster
- [ ] Redirecionamento HTTP → HTTPS habilitado globalmente via middleware
- [ ] Roteamento por subdomínio funcionando: `<app>.bisnaguete.xyz → serviço correto`
- [ ] Dashboard acessível em `traefik.bisnaguete.xyz` com HTTPS
- [ ] Integração com cert-manager validada (KB-007)
- [ ] Registros DNS criados no Cloudflare para todos os subdomínios ativos

## Configuração

| Parâmetro | Valor |
|-----------|-------|
| Versão | Traefik v3 (chart `traefik/traefik` — versão fixada em `values.yaml`) |
| Modelo de roteamento | `kind: Ingress` padrão do Kubernetes |
| HTTP → HTTPS | Redirecionamento global via entrypoint `web → websecure` |
| Dashboard | `traefik.bisnaguete.xyz` com HTTPS e autenticação básica |
| Namespace | `traefik` |

## Subdomínios do Cluster

Registros A a criar no Cloudflare (proxy desligado, apontando para `192.168.31.55`):

| Subdomínio | App |
|-----------|-----|
| `traefik.bisnaguete.xyz` | Dashboard do Traefik |
| `argocd.bisnaguete.xyz` | ArgoCD UI |
| `home.bisnaguete.xyz` | Glance |
| `pgadmin.bisnaguete.xyz` | pgAdmin |

> Novos subdomínios são adicionados aqui conforme novos apps são criados.

## Estrutura no Repositório

```
argocd/apps/
└── traefik.yaml              # Application CRD (wave 1)

apps/traefik/

```

## Notas Técnicas

- `--disable=traefik` e `--disable=servicelb` já configurados no Ansible (KB-002)
- O Traefik usa `hostNetwork: false` — exposto via `LoadBalancer` service no IP do node
- Autenticação do dashboard via middleware BasicAuth (senha em SealedSecret)

## Dependências

- KB-002 (Traefik nativo do K3s desabilitado no Ansible)
- KB-004 (App of Apps — wave 1)
- KB-007 (cert-manager para TLS — instalado na mesma wave)
- KB-008 (Sealed Secrets — para a senha do dashboard)
