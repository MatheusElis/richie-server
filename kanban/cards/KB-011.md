## Status
- **Coluna atual:** To Do
- **Prioridade:** Alta

## Descrição

Runbook completo para executar o primeiro setup do cluster do zero. Este card documenta todos os passos manuais que precisam ser feitos **uma única vez** antes de qualquer `make install` subsequente ser suficiente.

Após completar este card, rebuilds futuros se resumem a:
```bash
make install
```

## Pré-requisitos Externos (fora do repositório)

Antes de iniciar, confirmar que:

- [ ] Domínio `bisnaguete.xyz` ativo no painel do Cloudflare
- [ ] Registro DNS wildcard criado no Cloudflare:
  - Tipo: `A`
  - Nome: `*`
  - Conteúdo: `192.168.31.55`
  - Proxy: **DNS only** (nuvem cinza — obrigatório para DNS-01)
- [ ] Cloudflare API Token criado com permissões:
  - `Zone → DNS → Edit`
  - `Zone → Zone → Read`
  - `Zone Resources → bisnaguete.xyz`
  - Token salvo em local seguro (gerenciador de senhas)
- [ ] Ansible instalado na máquina (`pip3 install ansible`) ou deixar o `make deps` cuidar disso

## Passo a Passo do Primeiro Setup

### 1. Clonar o repositório

```bash
git clone https://github.com/MatheusElis/richie-server
cd richie-server
```

### 2. Instalar dependências

```bash
make deps
```

Instala automaticamente: `ansible`, `helm`, `kubeseal`.

### 3. Gerar a chave do Sealed Secrets

```bash
make gen-keys
```

Gera o par de chaves RSA e salva em:
- `~/.homelab/sealed-secrets-key.yaml` — chave privada (NUNCA no Git)
- `~/.homelab/sealed-secrets-cert.pem` — certificado público (usado para encriptar secrets)

> ⚠️ **Faça backup imediato da chave privada** em um gerenciador de senhas (Bitwarden, etc.) ou pendrive criptografado. Se perder, precisará rodar `make rotate-keys` e re-encriptar todos os secrets.

### 4. Encriptar o token do Cloudflare

```bash
kubectl create secret generic cloudflare-api-token \
  --namespace cert-manager \
  --from-literal=api-token=SEU_TOKEN_AQUI \
  --dry-run=client -o yaml \
  | kubeseal --cert ~/.homelab/sealed-secrets-cert.pem \
  -o yaml > apps/cert-manager/sealed-secret.yaml
```

> **Nota:** o `kubectl` não precisa de cluster ativo — o `--dry-run=client` gera o YAML localmente, e o `kubeseal --cert` encripta usando apenas o certificado público.

```bash
git add apps/cert-manager/sealed-secret.yaml
git commit -m "feat: adiciona sealed secret do cloudflare api token"
git push
```

### 5. Executar o setup completo

```bash
make install
```

O que acontece internamente:
```
make install
  └── make deps          # verifica ferramentas
  └── Ansible site.yml
      ├── role common    # OS: swap, sysctl, timezone, /opt/homelab/
      ├── role k3s       # instala K3s v1.32.3+k3s1
      ├── role sealed-secrets  # importa ~/.homelab/sealed-secrets-key.yaml no cluster
      └── role argocd    # instala ArgoCD via Helm + aplica root-app
          └── ArgoCD sincroniza App of Apps:
              wave 0: sealed-secrets controller
              wave 1: traefik + cert-manager
              wave 2: postgresql + pgadmin
              wave 3: glance + demais apps
```

### 6. Acompanhar a sincronização do ArgoCD

```bash
# Ver status dos pods do ArgoCD
kubectl get pods -n argocd

# Ver status das applications
kubectl get applications -n argocd

# Acompanhar logs do ArgoCD
kubectl logs -n argocd -l app.kubernetes.io/name=argocd-application-controller -f
```

### 7. Aguardar emissão do certificado TLS

O cert-manager pode levar alguns minutos para emitir o certificado wildcard via DNS-01:

```bash
# Verificar status do certificado
kubectl get certificate -n cert-manager

# Ver detalhes (útil para debugar)
kubectl describe certificate wildcard-bisnaguete-xyz -n cert-manager

# Ver challenges do ACME (deve aparecer e desaparecer automaticamente)
kubectl get challenges -n cert-manager
```

### 8. Validar os serviços

Adicionar ao `/etc/hosts` da máquina cliente (ou configurar no roteador):
```
192.168.31.55  home.bisnaguete.xyz
192.168.31.55  traefik.bisnaguete.xyz
192.168.31.55  pgadmin.bisnaguete.xyz
```

Testar:
- [ ] https://home.bisnaguete.xyz — Glance respondendo
- [ ] https://traefik.bisnaguete.xyz — Dashboard do Traefik
- [ ] https://pgadmin.bisnaguete.xyz — pgAdmin (login: `admin@bisnaguete.xyz`)
- [ ] Certificado TLS válido (sem aviso de segurança no browser)
