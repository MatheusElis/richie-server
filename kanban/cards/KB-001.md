## Estrutura Definida

```
richie-server/
├── Makefile                        # Ponto de entrada único (make install, make help, etc.)
├── README.md                       # Índice geral com links para docs/
│
├── ansible/                        # Configuração do OS e bootstrap do K3s
│   ├── inventory/                  # Hosts e variáveis de ambiente
│   ├── roles/                      # Roles reutilizáveis
│   └── playbooks/                  # Playbooks principais
│
├── argocd/                         # Tudo relacionado ao ArgoCD
│   ├── bootstrap/                  # Manifesto inicial (root-app) — aplicado 1x pelo Ansible
│   └── apps/                       # Application CRDs — um arquivo por app
│       ├── glance.yaml
│       ├── traefik.yaml
│       └── ...
│
├── apps/                           # Conteúdo de cada app (values Helm, ConfigMaps, manifests)
│   ├── glance/
│   │   ├── values.yaml
│   │   ├── configmap.yaml
│   │   └── sealed-secret.yaml      # Secret exclusivo do app fica aqui
│   ├── traefik/
│   │   └── values.yaml
│   └── ...
│
├── secrets/                        # SealedSecrets compartilhados entre múltiplos apps
│   └── (vazio inicialmente)
│
└── docs/                           # Documentação técnica
    ├── arquitetura.md              # Visão geral da stack e decisões técnicas
    ├── acesso.md                   # Como acessar os serviços localmente
    ├── como-adicionar-app.md       # Passo a passo para adicionar novo app via GitOps
    ├── sealed-secrets.md           # Como criar e gerenciar secrets
    └── hardware.md                 # Especificações do hardware
```

## Convenções Definidas

### Apps
- Cada app tem **dois lugares** no repositório:
  - `argocd/apps/<app>.yaml` → define o `Application` CRD do ArgoCD
  - `apps/<app>/` → contém o conteúdo do app (values, configmaps, etc.)
- Adicionar app = criar os dois. Remover app = deletar os dois.

### Secrets
- Secret **exclusivo de um app** → fica em `apps/<app>/sealed-secret.yaml`
- Secret **compartilhado entre múltiplos apps** → fica em `secrets/`
- **Nenhum secret em texto plano entra no repositório**

### Documentação
- `README.md` na raiz é o índice principal com links para `docs/`
- Documentação técnica detalhada fica em `docs/`
- Não há `README.md` por subpasta — toda documentação vai em `docs/`

## Notas Técnicas

- A estrutura é pensada para escalar: adicionar um novo node no futuro só impacta `ansible/inventory/`
- O ArgoCD aponta para `apps/<app>/` como source de cada Application
