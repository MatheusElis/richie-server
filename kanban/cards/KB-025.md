# KB-025 — Integrar ArgoCD com Authentik (OIDC)

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Média
- **Criado em:** 2026-04-22

## Descrição

Configurar SSO nativo do ArgoCD via OIDC com o Authentik. Permite login no ArgoCD usando as credenciais do Authentik, eliminando a necessidade da senha admin separada.

## Detalhes Técnicos

### Configuração no Authentik

1. Criar **Provider** tipo OAuth2/OpenID Connect
   - Client ID: `argocd`
   - Redirect URIs: `https://argocd.bisnaguete.xyz/auth/callback`
   - Signing Key: selecionar o certificado do Authentik
2. Criar **Application** vinculada ao provider
3. Mapear grupos do Authentik para roles do ArgoCD (ex: grupo `argocd-admins` → role `admin`)

### Configuração no ArgoCD

Atualizar `argocd/bootstrap/values.yaml` para incluir OIDC:

```yaml
configs:
  cm:
    url: https://argocd.bisnaguete.xyz
    oidc.config: |
      name: Authentik
      issuer: https://auth.bisnaguete.xyz/application/o/argocd/
      clientID: argocd
      clientSecret: $oidc.authentik.clientSecret
      requestedScopes:
        - openid
        - profile
        - email
  rbac:
    policy.csv: |
      g, argocd-admins, role:admin
```

### Manifests a alterar

- `argocd/bootstrap/values.yaml` (adicionar OIDC config)
- `secrets/argocd-oidc-secret.yaml` (SealedSecret com clientSecret do Authentik)

## Critérios de Aceite

- [ ] Login no ArgoCD via botão "Login with Authentik"
- [ ] SSO funcional — usuário do Authentik acessa o ArgoCD sem senha separada
- [ ] Mapeamento de grupo → role funcional (admins do Authentik = admins do ArgoCD)
- [ ] Login admin local continua funcionando como fallback

## Notas Técnicas

- O OIDC do ArgoCD é nativo (via Dex embutido ou direto), não precisa de Forward Auth
- Manter login admin local como fallback em caso de problemas com Authentik
- O clientSecret é armazenado como SealedSecret e referenciado no values.yaml

## Dependências
