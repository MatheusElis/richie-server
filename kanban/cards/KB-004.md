# KB-004 — ArgoCD App of Apps

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-20

## Descrição

Criar a estrutura base do padrão App of Apps no ArgoCD. Define como os aplicativos são declarados, organizados, ordenados e sincronizados no cluster via GitOps. O ArgoCD gerencia inclusive a própria instalação após o bootstrap.

## Critérios de Aceite

- [ ] App raiz (`root-app`) criada, apontando para `argocd/apps/`
- [ ] ArgoCD sincronizando automaticamente ao detectar mudanças no Git
- [ ] ArgoCD gerenciando sua própria instalação via App of Apps
- [ ] `syncPolicy` configurada com `automated + prune + selfHeal` em todos os apps
- [ ] Cada app rodando em seu próprio namespace isolado
- [ ] Sync waves configuradas respeitando a ordem de dependência
- [ ] Validado: adicionar arquivo em `argocd/apps/` → ArgoCD detecta e sincroniza
- [ ] Validado: remover arquivo de `argocd/apps/` → ArgoCD remove o app e recursos do cluster
- [ ] Documentação do padrão em `docs/como-adicionar-app.md`

## Sync Policy Padrão

Todos os apps usam:

```yaml
syncPolicy:
  automated:
    prune: true       # recursos removidos do Git são removidos do cluster
    selfHeal: true    # mudanças manuais no cluster são revertidas para o estado do Git
  syncOptions:
    - CreateNamespace=true  # cria o namespace automaticamente se não existir
```

## Namespaces

- **Um namespace por app** — isolamento total, remoção limpa, padrão da indústria
- Namespace criado automaticamente pelo ArgoCD via `CreateNamespace=true`
- Convenção: namespace = nome do app (ex: app `glance` → namespace `glance`)

## Sync Waves — Ordem de Instalação

Controla a ordem em que o ArgoCD instala os apps para respeitar dependências:

| Wave | Apps | Motivo |
|------|------|--------|
| `0` | `sealed-secrets` | Precisa estar pronto antes de qualquer SealedSecret ser aplicado |
| `1` | `traefik`, `cert-manager` | Infra de rede e TLS antes dos apps |
| `2` | `argocd`, `postgresql` | ArgoCD auto-gerenciado e banco antes dos apps dependentes |
| `3` | `glance` e demais apps | Apps finais, dependem de toda a infra |

```yaml
# Exemplo de anotação em argocd/apps/glance.yaml
metadata:
  annotations:
    argocd.argoproj.io/sync-wave: "3"
```

## Estrutura no Repositório

```
argocd/
├── bootstrap/
│   ├── root-app.yaml              # Aplicado 1x pelo Ansible — aponta para argocd/apps/
│   ├── values.yaml                # Helm values do ArgoCD
│   └── argocd-admin-secret.yaml   # SealedSecret da senha admin
└── apps/
    ├── argocd.yaml                # ArgoCD gerenciando ele mesmo (wave 2)
    ├── sealed-secrets.yaml        # (wave 0)
    ├── traefik.yaml               # (wave 1)
    ├── cert-manager.yaml          # (wave 1)
    ├── postgresql.yaml            # (wave 2)
    └── glance.yaml                # (wave 3)

apps/
├── glance/
│   └── values.yaml
├── traefik/
│   └── values.yaml
└── ...
```
