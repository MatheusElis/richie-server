# KB-017 — Radarr (gerenciador de filmes)

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Média
- **Criado em:** 2026-04-21

## Descrição

Deploy do Radarr para gerenciamento automático de filmes.

## Detalhes Técnicos

| Campo | Valor |
|---|---|
| Imagem | `linuxserver/radarr:6.1.1` |
| Porta | 7878 |
| Volumes | `/config` (persistente), `/movies` → `/opt/homelab/data/media/movies`, `/downloads` → `/opt/homelab/data/media/downloads` |
| Env vars | `PUID`, `PGID`, `TZ`, `RADARR__AUTH__METHOD=External`, `RADARR__AUTH__REQUIRED=DisabledForLocalAddresses` |
| Ingress | `https://radarr.bisnaguete.xyz` |

### Manifests

- `apps/radarr/deployment.yaml`
- `apps/radarr/service.yaml`
- `apps/radarr/ingress.yaml`
- `apps/radarr/pv.yaml` (config)
- `argocd/apps/radarr.yaml`

## Critérios de Aceite

- [ ] Pod running no namespace `radarr`
- [ ] Web UI acessível via `https://radarr.bisnaguete.xyz`
- [ ] Consegue ver pasta de filmes e downloads
- [ ] ArgoCD: Synced + Healthy

## Notas Técnicas

- **Autenticação:** `AuthenticationMethod=External` — confia no header `Remote-User` do Authentik. Sem tela de login própria. Annotation no ingress: `traefik.ingress.kubernetes.io/router.middlewares: authentik-authentik@kubernetescrd`
- **Integração com Prowlarr:** após deploy, configurar Prowlarr como gerenciador de indexadores via Settings → Indexers na web UI do Radarr. Essa configuração será feita juntos quando o card estiver em Review.

## Dependências

- KB-014 (volume compartilhado de mídia)
- KB-024 (Authentik — Forward Auth + header Remote-User)
