# KB-005 — Makefile: Ponto de Entrada `make install`

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-20

## Descrição

Criar o Makefile que serve como ponto de entrada único do projeto. O objetivo é que qualquer pessoa consiga recriar toda a infraestrutura do zero com um único comando, sem pré-requisitos manuais além do `make`.

```bash
git clone https://github.com/MatheusElis/richie-server
cd richie-server
make install  # instala deps + sobe o cluster completo
```

## Critérios de Aceite

- [ ] `make install` executa o fluxo completo do zero
- [ ] `make deps` verifica e instala automaticamente todas as ferramentas necessárias
- [ ] `make install` chama `make deps` antes de qualquer coisa
- [ ] `make reset` derruba e recria o cluster em um único comando
- [ ] `make uninstall` remove o cluster preservando `~/.homelab/`
- [ ] `make help` lista todos os targets com descrição
- [ ] Targets individuais funcionam de forma isolada
- [ ] Testado em máquina limpa (Ubuntu 24.04)

## Targets

| Target | Descrição |
|--------|-----------|
| `make install` | Fluxo completo: `deps → os → k3s → argocd` |
| `make deps` | Verifica e instala ferramentas faltando automaticamente |
| `make os` | Só configura o OS (role `common` do Ansible) |
| `make k3s` | Só instala/configura o K3s |
| `make argocd` | Só faz bootstrap do ArgoCD |
| `make uninstall` | Remove K3s e cluster (preserva `~/.homelab/`) |
| `make reset` | `uninstall` + `install` — rebuild completo em um comando |
| `make gen-keys` | Gera o par de chaves do Sealed Secrets (executado uma vez no primeiro setup) |
| `make rotate-keys` | Gera novas chaves e re-encripta todos os SealedSecrets automaticamente |
| `make help` | Lista todos os targets com descrição |

## Target `make deps` — Instalação Automática

Verifica e instala cada ferramenta se não estiver presente:

| Ferramenta | Método de instalação |
|---|---|
| `ansible` | `pip3 install ansible` |
| `helm` | Script oficial do Helm (`get-helm-3`) |
| `kubectl` | Symlink gerado pelo K3s — não instala separado |
| `kubeseal` | Download do binário via GitHub Releases |

## Target `make uninstall` — O que preserva

```
Remove:
  ✅ K3s e todos os seus componentes (k3s-uninstall.sh)
  ✅ Interfaces de rede residuais (flannel, cni0)
  ✅ ~/.kube/config

Preserva:
  🔒 ~/.homelab/                    # chave do Sealed Secrets, dados críticos
```

## Notas Técnicas

- O Makefile **não contém lógica complexa** — delega tudo para os playbooks Ansible
- Cada target Ansible recebe apenas as roles necessárias (ex: `make k3s` roda só a role `k3s`)
- Variáveis configuráveis via arquivo `.env` na raiz (não versionado) ou via `all.yml`
- `make help` gerado automaticamente a partir dos comentários dos targets (`## descrição`)
