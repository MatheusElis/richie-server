# KB-003 — Bootstrap do ArgoCD

## Status
- **Coluna atual:** Backlog
- **Prioridade:** Alta
- **Criado em:** 2026-04-20

## Descrição

Instalar o ArgoCD no cluster K3s via Ansible como etapa final do `make install`. O ArgoCD é instalado via Helm chart, com senha de admin fixa gerenciada por SealedSecret. Após essa etapa, o ArgoCD assume o controle via GitOps e tudo mais é gerenciado por ele.

O servidor é headless — toda validação é feita via CLI e logs, sem UI. A UI do ArgoCD só é acessada após o Traefik (KB-006) estar configurado via App of Apps.

## Critérios de Aceite

- [ ] Helm instalado na máquina (pré-requisito do playbook)
- [ ] Namespace `argocd` criado
- [ ] ArgoCD instalado via Helm chart `argo/argo-cd` versão `v2.14.x`
- [ ] Senha do admin definida via SealedSecret (fixa entre rebuilds)
- [ ] Repositório Git configurado como fonte no ArgoCD
- [ ] App raiz (root-app) aplicada e sincronizando
- [ ] Ansible valida automaticamente ao final: pods healthy, app list respondendo
- [ ] Idempotente: segunda execução não reinstala nem quebra nada

## Método de Instalação

```
Ansible
  └── helm install argo/argo-cd -n argocd -f argocd/bootstrap/values.yaml
      └── kubectl apply -f argocd/bootstrap/root-app.yaml
          └── ArgoCD sincroniza App of Apps → cluster se auto-configura
```

## Configuração de Acesso

- **Durante o bootstrap:** sem UI — validação exclusivamente via CLI e logs
- **Após KB-006 (Traefik):** UI acessível em `argocd.seudominio.com` com HTTPS

### Comandos de validação (executados pelo Ansible ao final)

| Comando | O que valida |
|---------|-------------|
| `kubectl get pods -n argocd` | Todos os pods em estado Running |
| `argocd app list` | Apps registradas e status de sync |
| `kubectl get applications -n argocd` | Estado dos CRDs |

O Ansible falha (`failed_when`) se alguma validação não passar — `make install` só retorna sucesso quando tudo estiver healthy.

## Senha do Admin

- Senha fixa definida pelo usuário, encriptada como **SealedSecret** e versionada no Git
- A mesma senha é mantida em todos os rebuilds — sem necessidade de consultar arquivos após cada `make install`
- O SealedSecret da senha é aplicado **antes** da instalação do ArgoCD pelo Ansible

## Variáveis Configuráveis (`all.yml`)

```yaml
argocd_version: "2.14"          # versão fixada, atualizável manualmente
argocd_namespace: "argocd"
argocd_repo_url: "https://github.com/MatheusElis/richie-server"
```

## Estrutura no Repositório

```
argocd/
└── bootstrap/
    ├── values.yaml              # Valores do Helm chart do ArgoCD
    ├── root-app.yaml            # Application CRD — aponta para argocd/apps/
    └── argocd-admin-secret.yaml # SealedSecret com a senha do admin
```
